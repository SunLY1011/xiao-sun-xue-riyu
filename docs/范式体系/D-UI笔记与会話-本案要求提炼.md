## 一、UI 界面美化原则（我们强调过的）

### 1.1 总原则

| 原则 | 要求 | 目的 |
|------|------|------|
| **学习优先** | 一屏一个主判断；次要信息折叠 | 降低认知负荷 |
| **平面轻量** | 无重阴影/渐变（R8） | WebView 性能、少「App 壳套壳」 |
| **高度预算** | `100dvh − 顶栏 − 底栏`（R3） | 底栏固定、内容区滚 |
| **390×844 铁律** | 切换页外壳尺寸不变（R12） | 与微信一致 |
| **图例即语义** | 书/人/あ 线性 SVG | 少文字猜测 |
| **日语优先** | 主标题日文；中文辅 | 学习场景 |
| **编码安全** | UTF-8；禁 PS 无 BOM 批量改中文（R9） | 防乱码毁页 |

### 1.2 两层 IA（开机 vs 地图）

| 层 | 是什么 | 不要什么 |
|----|--------|----------|
| **P0 开机** | 品牌 + 地图主视觉 + **开始学习** 单钮 | 24 课列表、海星统计、自动 1.5s 跳 |
| **P1 地图** | 四岛路径 + 单元折叠 + 00 入門紫条 | 课号超市、首页大下拉 |

底栏「课文」= 永远回 **P1 地图**，不是开机页。

### 1.3 课内五关 UI

| 项 | 定案 |
|----|------|
| 提示 | 「提示：」；**单行全文**无 ▾；多行才折叠 |
| 展开 | **日文在上、中文在下**（单词/文法/作业/扩展） |
| 底栏 | **全 App 唯一** 三钮：课文·笔记·注音（R11） |
| 换关 | 顶栏 Tab；**无**课内第二底栏、无传送链副本 |

### 1.4 刷新前 30 秒（R1–R13）

改 UI / 声称修好 / 硬刷新前 **必读** 汇编 §0 — 见索引 [00](./00-开发到收官-范式索引.md) §四。

## 二、会話（dialogue）为什么多轮修改 · 我们学到了什么

### 2.1 根因：会話不是「一个模块」，是四条产品线

| 子系统 | 内容 | 改动的敏感性 |
|--------|------|--------------|
| **课文句** | opener + A 标准答 | 对齐标日 · TTS 长链 |
| **ABC 变体** | B/C 日文可见不同 + 场景说明第三行 | 产品真源 · 376 场景 |
| **交互壳** | 场景折叠、三卡、🔊🎤▶、按住录音 | 微信录音权限 |
| **评语** | 句下全宽；换句清空 | 记忆库 §3 反复 |

任何「改会話 UI」都可能误触 **ABC 语义** — 所以我们后来 **§4.1 冻结**。

### 2.2 迭代时间线（摘要）

| 阶段 | 我们要求什么 | 为什么 |
|------|--------------|--------|
| 早期 | 跟读 🔊🎤 · 场景折叠 | 听说闭环 |
| U1 冻结 | L1 五关 + ABC 结构锁 | 种子课不可被批量脚本冲掉 |
| v58–v63 | 少文字 · 按住录 · 评语生命周期 | 微信真机 |
| ABC 全课 | A=课文 B/C=变体 **三行可见** | 学员要能对比语体，不能藏折叠里 |
| audit 驱动 | `abc_variants.py` · vis(B)≠vis(A) | 机器验「B/C 真不同」 |
| v408 | 376 场景黄卡 · TTS 全覆盖 | 笔记「老师·会话要点」同源 |
| **用户冻结** | 「会話模块我没要求改」 | **壳可动，ABC 三行/三卡逻辑不动** |

真源：`会話-ABC三答-产品真源.md` · `绘画与会话ABC-全24课评估清单.md` · 汇编 §4.1

### 2.3 会話改动准入（以后）

| 允许 | 禁止（除非用户明示） |
|------|----------------------|
| 🔊 源顺序 · 按住录音 · 失败 toast | 删 B/C · 合并三行 · 改变 A/B/C 标签语义 |
| 评语清空逻辑 · TTS 镜像 | 改 `dialogue-gate` ABC 渲染规则 |
| 全局 dock/顶栏（不碰 ABC 区） | 未读 §4.1 就「优化会話 UI」 |

### 2.4 设计目的（发心）

- **A** = 课文标准答案（跟教室一致）  
- **B/C** = 同场景语体/礼貌度对比（**可见**才能学）  
- **录音** = 说一遍验一遍，不是装饰  
- **冻结** = 保护已验收教学结构，把迭代力放到 **数据深度/TTS/笔记联动**

## 三、笔记 / 知识库板块 · 我们做了哪些要求

### 3.1 定位

| 项 | 定案 |
|----|------|
| 名称 | 原「我的」→ **笔记**（底栏中间） |
| 角色 | **档案 + 三维知识** + 自笔记；不是第二套课内 |
| 与课文关系 | 关联知识 **弹窗不跳课** |

### 3.2 六维结构（每课手风琴）

| 块 | 内容 | 数据来源 |
|----|------|----------|
| 📖 课本 | 语法点+解释 | grammarNodes |
| 👨‍🏫 老师 | 发音/语源/敬语/会话要点… | summaryBlocks · **dialogueKeyPoints** |
| 🧠 学霸 | 核心句型/易错/参考 | 自动合成 |
| 🎯 薄弱 TOP3 | 本课错题排名 | mistakes 按 lessonId |
| 🔗 关联知识 | 概念链弹窗 | **knowledge-graph**（v399+） |
| ✏️ 我的笔记 | 自由文本 | localStorage |

### 3.3 UI 要求

| 要求 | 说明 |
|------|------|
| 主区最大化 | 计时+列表 = 主滚动；设置 **沉底 dock 默认折叠** |
| 手风琴单开 | 与单元规则一致 |
| 易错对照 | **红大字正解在上、蓝小字曾错在下** |
| 学霸块 | 禁 nested `<details>`（微信） |
| 六色块 | 对齐五关色板令牌 |
| V1 锁 | `lock/notes-panel-v1-20260525` — 无明示不改结构 |

真源：`过程讨论内容/19-笔记板块规范.md` · `docs/locks/NOTES-PANEL-V1-LOCK-20260525.md`

### 3.4 与 L1–24 内容层联动（v408–399）

| 联动 | 目的 |
|------|------|
| 954 单词黄卡 | 课内难词与笔记同源 |
| 101 GRAMMAR 块 | 文法关黄卡 = 笔记可检索 |
| dialogueKeyPoints | 会話要点进「老师」维 |
| knowledge-graph | 关联弹窗 = 跨课概念链，不破坏课序 IA |

## 四、知识库（项目级）与笔记的区别

| | 项目知识库 `docs/项目知识库-*` | 学员笔记 `#view-me` |
|--|-------------------------------|---------------------|
| 读者 | Agent / 作者 | 学员 |
| 内容 | 日文书写、TTS、课内单词范式、文递自归 | 课内数据汇总 + 自笔记 |
| 改动的意义 | 防 Agent 写错日语/破坏流程 | 防学员看不到薄弱/关联 |

改 **知识库** = 改协作纪律；改 **notes-panel.js** = 改学员档案 UI — 档位不同（L3+）。

## 五、优化改进时怎么用本文

```text
收到「改 UI / 改会話 / 改笔记」
  ↓
① 对齐发心（B 文档 §一）— 为学习还是为好看？
  ↓
② 查冻结 — 会話 §4.1？笔记 V1 锁？MVP 课内？
  ↓
③ 讨论优先 — 复述 + 待拍板
  ↓
④ 最小 diff + 真机壳目视 + pre-ship
  ↓
⑤ 拍板后写入 confirmed / 汇编 §
```
