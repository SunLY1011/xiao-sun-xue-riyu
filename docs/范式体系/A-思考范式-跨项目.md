# A · 思考范式（跨项目 · 可套用）

> **用途**：任何「作者 + Agent 协作做学习型软件」都适用的**元纪律**。  
> **本案落点**：Cursor 规则 `discuss-before-act` · `wendi-zigui-core` · `intepoint-panorama-governance`

---

## 1. 讨论优先（对齐意图 · 先于写码）

| 步骤 | 我们做什么 |
|------|------------|
| ① 接收 | 文字、截图、线框、语音转写 |
| ② 讨论 | Agent **先**复述理解、列待拍板、给方案对比 |
| ③ 确认 | 用户「可以开工 / 同意 / 按这个做」 |
| ④ 动作 | 再改代码、数据、样式 |

**例外**（须一句话说明）：用户点名只写文档；修复**已确认方案内**的笔误。

真源：`过程讨论内容/00-讨论优先铁律.md`

## 2. 文递自归（对齐基线 · 增量叠层）

**技术标准名**：Autoregressive Generation（自回归生成）

| 工程映射 | 含义 |
|----------|------|
| `confirmed[]` | 已生成、不可擅自推翻的上下文 |
| 本轮 diff | 下一 Token · 最小改动 |
| 禁止 | 整站重写、未读基线就改 |

**ICE 档位**（改前自评）：L0 只读 → L1 单文件 → L2 关内 → L3 四关+语音 → L4 已批方案 → L5 须逐条明示

**CONFIRM 六步**：C 对齐 → O 仅本轮 → N 最小 diff → F 修复+验证 → I 入账 JSON → R 交付反馈块

真源：`docs/Agent文递自归.md` · `docs/项目知识库-文递自归.md`

## 3. INTEPOINT 全景 Soul（先想清楚再落控件）

从 INTEPOINT 人才模型提炼的**产品思考维度**（非岗位标签）：

| 元组件 | 在学习 App 里是什么 |
|--------|---------------------|
| **领域池** | 这门课/这门语言有哪些能力块（如五十音 8 域） |
| **三层** | 打底 / 协同 / 核心 — 先广后深 |
| **作业法** | 固定步骤链（讨论→种子→冻结→wave→ship） |
| **图例即语义** | L0 图标一眼懂，不依赖底部小字猜 |
| **金句锚定** | 反复出现的原则句（学习优先、100% 才星） |

真源：`过程讨论内容/11-五十音INTEPOINT全景模型启发.md`

## 4. 「我们」协作范式

| 原则 | 做法 |
|------|------|
| 称谓 | **我们**，不是「您的产品」 |
| Agent 角色 | PM + 体验官 + 工程助理；**不**替代教师/作者判断 |
| 派单 | 全局布局/视觉 → `hyouga-product-pm` 先拆再 Dev |
| 交付 | 先证据（pre-ship）后结论；附双链接 + 真机框行 |

真源：`docs/PRODUCT-PM-全局导演手册.md` · `docs/我们-产品愿景-学习の道.md` §协作

## 5. 围绕发心不跑偏（修正而不偏离）

```text
发心（§B 产品原点）
  ↓ 每次需求用四句对齐
教学逻辑 → 产品逻辑 → 界面逻辑 → 实现边界
  ↓
改完对照：是否仍「学习优先 / 课序同构 / 满分才星」
  ↓
入账：拍板的进 confirmed；未做的进 backlog
```

**修正** = 在 confirmed 上叠 diff（文递自归）  
**偏离** = 未讨论就改 IA、删 Tab、改冻结会話 ABC、用 localhost 当发行
