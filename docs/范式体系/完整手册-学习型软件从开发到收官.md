## 1.2 核心思想

```text
发心（为什么做）
```

# 第二篇 · 思考范式

> **副标题**：任何学习型 App 都适用的 **元纪律**（学科可换，协作规则不变）。

---

## 2.1 讨论优先

**核心思想** · 新需求先对齐意图；用户说「可以开工 / 同意 / 按这个做」才动仓库。

**金句** · *先复述，后 diff；见字就改 = 误读成本高。*

| 步 | 动作 |
|----|------|
| ① 接收 | 文字、截图、线框 |
| ② 讨论 | 复述理解、列待拍板、方案对比 |
| ③ 确认 | 用户明示可开工 |
| ④ 动作 | 改代码、数据、样式 |

**例外**（须一句话说明）：用户点名只写文档；修复**已确认方案内**笔误。

**▶ 判断** · 新截图 / 新 UI → **必须先讨论**，无例外。

---

## 2.2 文递自归（Autoregressive Generation）

**核心思想** · 每一轮改动 = 在已验收上下文上叠一层；`iteration-baseline.json` → `confirmed[]` = 已生成 Token 队列；禁止整站重写。

**金句** · *未读 baseline 不动手；本轮 scope 仅用户原话；backlog 不得擅自提前做。*

### ICE 档位（改前自评）

| 档 | 名称 | 范围 | 必做验证 |
|----|------|------|----------|
| L0 | 只读 | 读基线、解释 | 无改码 |
| L1 | 单点 | 1 文件 bug | 相关 1 项冒烟 |
| L2 | 关内 | 同 Tab 2–4 文件 | 该关完整冒烟 |
| L3 | 四关+语音 | data / speech-engine / 多关 UI | pre-ship 全绿 + 四关 |
| L4 | 方案迭代 | 用户书面同意的 P0–P3 | L3 + 入账 confirmed |
| L5 | 禁止擅自 | 删功能、大重构、换 IA | **须逐条明示** |

### CONFIRM 六步

**C** 对齐基线 → **O** 仅本轮 scope → **N** 最小 diff 文件清单 → **F** 修复+验证 → **I** 入账 JSON → **R** 交付反馈块

---

## 2.3 INTEPOINT 全景 Soul

**核心思想** · 先想清楚能力全景，再落按钮和页面；用「领域池 + 三层 + 作业法」思考，不是堆控件。

**金句** · *图例即语义；先全景 Soul，再局部好看。*

| 元组件 | 在学习 App 里 |
|--------|----------------|
| 领域池 | 能力块清单（如五十音 8 域、24 课文法域） |
| 三层 | 打底 / 协同 / 核心 |
| 作业法 | 讨论→种子→冻结→wave→ship→close |
| 图例先行 | L0 图标一眼懂，不靠底部小字猜 |
| 金句锚定 | 学习优先、100% 才星 |

---

## 2.4 「我们」协作

**核心思想** · Agent = PM + 体验官，不替代老师/作者判断；称谓用「我们」。

**金句** · *先证据后结论；未 pre-ship 全 OK 不说「修好了」。*

全局产品/布局/视觉 → 先用 **hyouga-product-pm** 拆逻辑，再派 Dev 实现。

---

## 2.5 围绕发心不跑偏

**核心思想** · **修正** = 在 confirmed 上叠 diff；**偏离** = 未讨论改 IA、删 Tab、改冻结会話 ABC、用 localhost 当发行。

```text
用户表述
  ↓ 教学逻辑（先音表再开口）
  ↓ 产品逻辑（入門/地图/四关）
  ↓ 界面逻辑（一层导航一件事）
  ↓ 实现边界（动 catalog；不动冻结课内）
改完对照：仍「学习优先 / 课序同构 / 满分才星」？
```

# 第三篇 · 产品设计范式

> **副标题**：发心与骨架 — **学科可换，结构复用**。

---

## 3.1 四条发心（一切改进的出发点）

**核心思想** · 任何优化都回答：**帮学会，还是帮看起来热闹？**

**金句** · *学习优先 · 课本同构 · 我们共学 · 100% 才给星。*

| # | 原点 | 我们说过什么 | 改进时自问 |
|---|------|--------------|------------|
| 1 | **学习优先** | 主路径 = 听·说·读·测；趣味在**通关之后** | 有效练习次数增加了吗？ |
| 2 | **课本同构** | 课序与教材规划表一致；首页 = **路径/地图** | 还跟着课序吗？变超市了吗？ |
| 3 | **我们** | Agent = PM+体验官，不替代教师 | 拍板了吗？ |
| 4 | **满分才星** | 金星 = 100%；「点过」不算学会 | 星级被稀释了吗？ |

**本案定案** · **写** 不纳入海星评价。

**趣味边界** · 插画 / 彩蛋 / 漫画收藏 = **通关后奖励**，不替代四关；グルミ 绘本叙事，不另设「小李」漫画人设进条带。

---

## 3.2 三层信息架构

**核心思想** · 首页是「我在哪」，课内是「我怎么学」，笔记/マイ是「我学得怎样」。

**金句** · *层1 我在哪 → 层2 我怎么学 → 层3 我学得怎样。*

```text
层1  目录/地图     →  四岛路径、单元、00入門
层2  课内关卡      →  単語 · 会話 · 文法 · テスト
层3  督导/笔记     →  进度、薄弱、关联、自笔记
```

**禁止** · 8 课平铺成「全书已上线」超市。

**本案** · `curriculum-catalog.js` = 课序/released/地图 **唯一真源**。

---

## 3.3 种子 · 冻结 · 分波

**核心思想** · 先 1–3 课四关金标跑通 → tag 冻结内测 → 再 wave 批量；对外诚实分 **金/银/铜** 档。

**金句** · *种子冒烟 → 冻结内测 → 分波播种 → 分档诚实对外。*

| 阶段 | 做法 | 本案 |
|------|------|------|
| 种子 | 1–3 课四关冒烟 | L14/16/18 金标 |
| 冻结 | tag；课内不改 | `mvp-v1.0.1-biaori-141618` |
| 分波 | Wave A→D | v408 L1–24 内容层 |
| 解锁 | 用户明示 + baseline | `all_lessons_l1_24_content` unlocked |

---

## 3.4 游戏化边界

| 用 | 不用 |
|----|------|
| 路径节点、海星/海胆、隐藏 00、漫画收藏 | 未统一隐喻（萝卜） |
| 课毕插画 → 单元连张 → 故事书 | 未三星即给大奖 |
| 写 **永不** 进星级（本案） | 85 分及格冒充「学会」 |

---

## 3.5 UI 与界面美化原则

**核心思想** · 移动端学习 UI = 平面、高度预算、真机一致；不为桌面全宽炫技。

**金句** · *平面轻量 · 高度预算 · 390×844 不变 · 日语主中文辅 · 图例即语义。*

| 原则 | 要求 | 目的 |
|------|------|------|
| 平面轻量 | 无重阴影/渐变 | WebView 性能与可读 |
| 高度预算 | `100dvh − 顶栏 − 底栏` | 底栏固定、内容滚 |
| 真机铁律 | 390×844 壳，切页外壳**不变** | 与微信 WebView 一致 |
| 双层 IA | **P0 开机**（品牌+开始学习）≠ **P1 地图**（选课） | 两层首屏不混 |
| 底栏唯一 | 课文·笔记·注音；课内**无**第二底栏 | 导航不分裂 |
| 手风琴单开 | 单元/课次同时只展开一个 | 移动端扫视 |
| 日语优先 | 主标题日文，中文辅 | 学习场景 |
| 编码 | UTF-8；禁 PS 无 BOM 批量改中文 HTML | 防毁页 |

**双界面交付（本案铁律）**

| 界面 | 用途 | 入口 |
|------|------|------|
| ① 真机壳 | 改 UI / 交付目视 | `cursor-miniapp-phone.html?live=1` |
| ② 微信 https | 学员学习 | `index.html?v=CACHE_VER` 公网 |

**禁止** · 用 localhost / 桌面全宽单独声称「UI 已验收」。

---

## 3.6 四层逻辑拆解（收到任何需求必做）

**金句** · *先教学逻辑，再产品逻辑，再界面逻辑，最后才是文件路径。*

```text
用户表述
  ↓ 教学逻辑   「先音表地图，再分块，再开口」
  ↓ 产品逻辑   「入門=全景三层；首页=路径；课内=四关」
  ↓ 界面逻辑   「一层导航一件事；日语主中文辅」
  ↓ 实现边界   「动 catalog/journey；不动冻结课内/会話§4.1」
```

## 3.7 内容生产（PRD → 学员端）

**核心思想** · 教材/PRD 为真源 → 生成脚本 → data → audit → TTS → pre-ship → bump cache。

**金句** · *新增日文必有 MP3；编号脚本对账，不手查。*

## 4.2 发现 → 解决 → 评估（闭环）

**核心思想** · 问题归类到 ICE 档；评估靠脚本或目视，不靠口头「应该好了」。

**金句** · *根因优先 · 最小 diff · 脚本对账 · UTF-8 不毁中文。*

```text
发现 → 归类(ICE) → 最小修复 → 档位验证 → 评估 → 入账 confirmed 或 backlog
```

| 发现渠道 | 信号 | 动作 |
|----------|------|------|
| 用户 | 截图、原话 | 讨论优先，不默认改码 |
| 机器 | pre-ship `[FAIL]` | 修根因，再跑 |
| 审计 | ABC/TTS/日文/PRD | 读 `docs/*-最新.md` |
| 开发 | 白屏、控制台 | 8765 + 真机壳 |
| 双通道 | 壳 OK、微信不行 | 4G 公网链、`?v=`、TTS 源 |
| 回归 | 改 A 坏 B | L14 四关冒烟 |

| 评估层级 | 通过标准 |
|----------|----------|
| L1–L2 | 相关冒烟 OK |
| L3 | pre-ship 全 `[OK]` + 四关 |
| UI | 汇编 §0 R1–R13 + 壳 L3 |
| 发链 | pre-ship + 双通道 + 交付反馈块 |
| 收官 | 第六篇 checklist 全 ✅ |

## 4.3 意见收纳 · 宽度与平选标准

**核心思想** · 不是每句话都进 confirmed；铁律和拍板才入库。

**金句** · *可验收 · 有 since_cache · 有 files · 不冲突铁律 — 四条齐才进 confirmed。*

### 收纳宽度

| 类别 | 收？ | 落点 |
|------|------|------|
| 铁律 / 同一要求 ≥3 次 | ✅ | Cursor 规则 · 本手册 · confirmed |
| 用户截图 + 原话拍板 | ✅ | 纪要汇编 · confirmed |
| 一次性偏好 | ⚠️ | 仅会话 |
| 未拍板新 UI | ❌ | §待拍板 · backlog planned |
| Agent 脑补 | ❌ | 不收 |

### 进 `confirmed[]` 须全满足

1. 用户明示或纪要 ✅ 拍板  
2. 可验收（一句话 + 目视/脚本）  
3. `since_cache` 已知  
4. `files[]` 可列  
5. 不冲突更高铁律（讨论优先、冻结课内、会話 §4.1、笔记 V1 锁）

### 好意见 vs 差意见（Review 过滤）

| 好 | 差 |
|----|-----|
| 「CTA 下移 20px 不挡 02」 | 「再好看一点」 |
| 「评语只留当前句」+ 发心 | 「加更多动画」 |
| 「只改 splash.css」 | 「顺便重构 app.js」 |

## 4.4 七阶段 · 要求 · 产出 · 检验 · 阶段收官动作

**核心思想** · Phase 6 才是收官；**push ≠ 收官**。

**金句** · *机器绿 + 真机目视 + 报告 + tag = 收官。*

|-------|----------|----------|----------|--------------|
| **0 愿景** | 课序表、三层 IA、游戏化边界 | 愿景 md、纪要 | 团队复述一致 | 纪要 ✅ 拍板 |

| **2 冻结** | 课内不改 | git tag、MVP-FREEZE | confirmed 含冻结 id | `git tag` push |

| **4 体验** | 微信/录音/TTS/真机 | 补丁版 | 用户截图问题关闭 | v409–410 类 |

7. （可选）国内 Pages / CDN

## 5.2 会話 · 为何多轮修改 · 定案是什么

**核心思想** · 会話不是「一个模块」，是 **四条产品线**；改壳易误伤 ABC 教学语义，故 **§4.1 冻结**。

**金句** · *A=课文标准答；B/C=可见不同语体；第三行场景说明不能藏；壳可修，ABC 语义不动。*

### 四条产品线

| 线 | 内容 | 敏感性 |
|----|------|--------|
| 课文句 | opener + A 标准答 | 对齐标日 · TTS |
| ABC 变体 | B/C 日文**可见不同** + 第三行场景说明 | 产品真源 · 376 场景 audit |
| 交互壳 | 场景折叠、三卡、🔊🎤▶、按住录音 | 微信权限 |
| 评语 | 句下全宽；换句/换录音清空 | 记忆库反复 |

### 对话在讲什么（定案）

| 部分 | 含义 |
|------|------|
| 发起句 | 标日课文问话/话题 |
| **A** | 课文标准答：日文 + 中文两行 |
| **B / C** | 场景变体：日文 + 中文 + **第三行场景说明**（对谁、何场合、与 A/B/C 差异） |

**禁止** · 删 B/C；把第三行藏进只有点开才见的折叠。

### 迭代时间线（摘要）

| 阶段 | 我们坚持什么 | 为什么 |
|------|--------------|--------|
| 早期 | 🔊🎤 听说闭环 | 可验证「说」 |
| U1 冻结 | L1 五关 + ABC 结构 | 种子不可被批量冲掉 |
| 全课 ABC | B/C vis 不同于 A | 学员要能**对比语体** |
| v408 | 黄卡 + 笔记「老师·会话要点」同源 | 课内与会后档案一致 |
| 用户冻结 | 「会話模块我没要求改」 | 保护已验收教学结构 |

### ▶ 会話改动准入

| 允许 | 禁止（除非明示） |
|------|------------------|
| TTS 源/镜像、按住录音、评语清空、失败 toast | 删 B/C、改三行语义、改 A/B/C 标签含义 |
| 全局顶栏/dock（不碰 ABC 区） | 未读 §4.1 就「优化会話 UI」 |
