# 总览

## 副标题 · 这套东西管什么

把「从零 → L1–24 收官」里**说过很多遍、决定产品不跑偏**的东西，收成 **5 层 + 1 入口**。

**先对齐发心，再对齐基线，再最小改动；机器过了，人眼再过，才叫收官。**

## 金句

> **学习优先，趣味在后；讨论在先，写码在后；confirmed 之上叠 diff，禁止整站重来。**

# A · 思考范式（跨项目）

## 副标题 · 任何学习型 App 都适用的元纪律

不管教日语还是数学，**人和 Agent 怎么一起想、怎么一起改**，规则相同。

---

## A1 · 讨论优先

### 核心思想

**新需求先对齐意图，用户说「可以开工」才动仓库。**

### 金句

> **先复述，后 diff；见字就改 = 误读成本高。**

### 四步

接收 → 讨论（复述+待拍板）→ 确认 → 动作

### ▶ 请你判断

- 新截图/新 UI 想法：**是否必须先讨论？** → **是，铁律**

---

## A2 · 文递自归（增量基线）

### 核心思想

**每一轮改动 = 在已验收上下文上叠一层；`confirmed[]` 是已生成的 Token 队列。**

### 金句

> **未读 baseline 不动手；本轮 scope 仅用户原话；backlog 不得擅自提前做。**

### ICE 档位（改前自评）

| 档 | 范围 |
|----|------|
| L1 | 单文件 bug |
| L2 | 同一关 2–4 文件 |
| L3 | 四关+语音 → 须 pre-ship 全绿 |
| L4 | 用户已批方案 → 入账 confirmed |
| L5 | 删 Tab/换 IA → **须逐条明示** |

### CONFIRM 六步

**C** 对齐 → **O** 仅本轮 → **N** 最小 diff → **F** 修复验证 → **I** 入账 JSON → **R** 交付反馈块

---

## A3 · INTEPOINT 全景 Soul

### 核心思想

**先想清楚能力全景（领域池·三层·作业法），再落按钮和页面。**

### 金句

> **图例即语义；先全景 Soul，再局部好看。**

### 五元

领域池 · 三层（打底/协同/核心）· 作业法 · 金句锚定 · 图例先行

---

## A4 · 「我们」协作

### 核心思想

**Agent = PM + 体验官，不替代老师/作者判断；称谓用「我们」。**

### 金句

> **先证据后结论；未 pre-ship 全 OK 不说「修好了」。**

# B · 产品设计范式

## 副标题 · 发心与骨架（学科可换，结构复用）

---

## B1 · 四条发心（改进为了什么）

### 核心思想

**任何优化都要回答：这是在帮学会，还是在帮看起来热闹？**

### 金句

> **学习优先 · 课本同构 · 我们共学 · 100% 才给星。**

| # | 原点 | 改进时自问 |
|---|------|------------|
| 1 | 学习优先 | 有效练习次数增加了吗？ |
| 2 | 课本同构 | 还跟着教材课序吗？ |
| 3 | 我们 | 拍板了吗？ |
| 4 | 100% 才星 | 点过算学会了吗？（不应） |

### 趣味边界

插画/彩蛋/漫画 = **通关后奖励**，不替代四关。

---

## B2 · 三层信息架构

### 核心思想

**首页是路径不是超市；课内是关卡；笔记/マイ是督导。**

### 金句

> **层1 我在哪 → 层2 我怎么学 → 层3 我学得怎样。**

### 禁止

8 课平铺成「全书已上线」。

---

## B3 · 种子 · 冻结 · 分波

### 核心思想

**先 1–3 课金标跑通，再 tag 冻结，再 wave 批量；避免「24 课同深度幻觉」。**

### 金句

> **种子冒烟 → 冻结内测 → 分波播种 → 金/银/铜分档诚实对外。**

---

## B4 · UI 美化六原则

### 核心思想

**移动端学习 UI = 平面、预算、真机一致；不为桌面炫技。**

### 金句

> **平面轻量 · 高度预算 · 390×844 不变 · 日语主中文辅 · 图例即语义。**

| 原则 | 一句话 |
|------|--------|
| 平面轻量 | 无重阴影渐变 |
| 高度预算 | 100dvh − 顶栏 − 底栏 |
| 真机铁律 | 切页外壳尺寸不变 |
| 双层 IA | P0 开机 ≠ P1 地图 |
| 底栏唯一 | 课文·笔记·注音，课内无第二底栏 |

# C · 工程交付范式

## 副标题 · 材料、问题、意见、收官

---

## C1 · 材料从哪来

### 核心思想

**别靠会话记忆；真源在文件里，改前先打开对的那一本。**

### 金句

> **意图看消息，已验收看 confirmed，UI 看汇编 §0，课序看 catalog，踩坑看记忆库。**

| 要找 | 打开 |
|------|------|
| 已拍板 | `iteration-baseline.json` → confirmed |
| 未做 | backlog |
| UI 目视 | 会议纪要汇编 §0 R1–R13 |
| 机器问题 | `发布前自检.bat` |
| 会話定案 | 会話-ABC三答-产品真源 |

---

## C2 · 发现 → 解决 → 评估

### 核心思想

**问题要归类到 ICE 档位；评估要有脚本或目视，不能口头「应该好了」。**

### 金句

> **根因优先 · 最小 diff · 脚本对账 · UTF-8 不毁中文。**

| 发现渠道 | 典型工具 |
|----------|----------|
| 用户截图 | 讨论优先 |
| 机器 FAIL | pre-ship |
| ABC/TTS/日文 | audit-*.py |
| 壳 OK 微信不行 | 4G 公网链 + ?v= |

---

## C3 · 意见收纳 · 宽度与平选

### 核心思想

**不是每句话都进 confirmed；铁律和拍板才入库，脑补和一次性偏好不进。**

### 金句

> **可验收 · 有 since_cache · 有 files · 不冲突铁律 — 四条齐才进 confirmed。**

| 收纳 | 落点 |
|------|------|
| ✅ 铁律 / 反复≥3次 | Cursor 规则 + 范式 |
| ✅ 截图+拍板 | 纪要汇编 |
| ⚠️ 一次性偏好 | 仅会话 |
| ❌ 未拍板新 UI | §待拍板 |
| ❌ Agent 脑补 | 不收 |

---

## C4 · 七阶段 · 要求与收官动作

### 核心思想

**每个阶段有「产出物」和「检验标准」；Phase 6 才是收官，不是 push 就算。**

### 金句

> **机器绿 + 真机目视 + 报告 + tag = 收官；缺一项只叫「已 push」。**

| Phase | 我们要求 | 检验 |
|-------|----------|------|
| 0 愿景 | 课序+IA+游戏化边界 | 纪要拍板 |
| 1 种子 | 1–3课四关冒烟 | 无白屏 |
| 2 冻结 | tag，课内不改 | MVP-FREEZE |
| 3 播种 | wave+audit+TTS0 | pre-ship 绿 |
| 4 体验 | 微信/录音/TTS链 | 截图问题关 |
| 5 双通道 | 壳+https 同 v | R1–R13 |
| 6 收官 | 报告+清单+tag | E 文档 checklist |

# D · UI · 会話 · 笔记

## 副标题 · 本案反复改过的三块 · 为什么

---

## D1 · UI · 刷新前必对（R1–R13 精华）

### 核心思想

**改 UI 声称 OK 前，30 秒清单不过 = 不能交付。**

### 金句

> **底栏三钮完整 · 笔记主区滚 · 设置沉底 · 壳 390×844 不变 · 会話未要求不改。**

---

## D2 · 会話 · 为何改了很多轮

### 核心思想

**会話 = 课文句 + ABC 变体 + 交互壳 + 评语四条线；改壳容易误伤 ABC，故 §4.1 冻结。**

### 金句

> **A=课文标准答；B/C=可见不同语体；第三行场景说明不能藏；壳可修，ABC 语义不动。**

### 迭代摘要

| 阶段 | 我们坚持 |
|------|----------|
| 早期 | 🔊🎤 听说闭环 |
| U1 冻结 | L1 五关+ABC 结构 |
| 全课 ABC | B/C 日文 **可见不同**（376 场景 audit） |
| v408 | 黄卡+笔记「老师·会话要点」同源 |
| 用户冻结 | **「会話模块我没要求改」** |

### ▶ 请你判断

- 想「优化会話 UI」→ **先读 §4.1，是否动 ABC 三行/三卡？** 动则须你明示。

---

## D3 · 笔记 · 六维要求

### 核心思想

**笔记 = 档案+三维知识+自笔记；关联弹窗不跳课；V1 已锁结构。**

### 金句

> **课本·老师·学霸·薄弱·关联·自笔记；易错红上蓝下；设置沉底默认折叠。**

| 块 | 一句话 |
|----|--------|
| 📖 课本 | 语法点+解释 |
| 👨‍🏫 老师 | 含会話要点 dialogueKeyPoints |
| 🧠 学霸 | 自动合成 |
| 🎯 薄弱 | 本课错题 TOP3 |
| 🔗 关联 | knowledge-graph 弹窗 |
| ✏️ 自笔记 | localStorage |

---

## D4 · 优化改进四问（不跑偏）

### 核心思想

**收到改 UI/会話/笔记，用四问过滤再讨论。**

### 金句

> **为学习？已冻结？讨论了吗？最小 diff？**

```text
① 对齐发心（B1）
② 查冻结（会話§4.1 / 笔记V1 / MVP课内）
③ 讨论优先
④ 真机壳 + pre-ship
```

# 拍板速查 · 最常要的三个判断

## 1 · 能不能改会話？

| 情况 | 答案 |
|------|------|
| 修 TTS/录音/评语清空 | ✅ 通常可以 |
| 改 ABC 三行/删 B/C | ❌ 须你明示 |
| 改场景折叠壳（不动 ABC 区） | ⚠️ 讨论后 |

## 2 · 能不能改笔记结构？

| 情况 | 答案 |
|------|------|
| 数据深度（黄卡/关联） | ✅ v408 已做 |
| 改六维布局/V1 锁 | ❌ 须你明示 |

## 3 · 能不能声称「可发学员」？

| 条件 | 必须 |
|------|------|
| pre-ship | 全 OK |
| 链接 | 含 ?v=410 |
| 目视 | 真机壳 + 建议 4G |
| 话术 | 不用 localhost |
